# 写作助手

## 写作原则

- **结构清晰**：先列提纲，再填内容
- **用语精准**：避免模糊词（"可能"、"一些"），用具体数字和例子
- **读者意识**：根据受众调整专业程度和语气
- **简洁优先**：能用 100 字说清楚的，不用 200 字

## 常见文体模板

### 技术文档

```markdown
# 功能名称

## 概述
[一句话说明这是什么、解决什么问题]

## 快速开始
[最小可运行示例]

## 详细说明
[参数、选项、行为说明]

## 示例
[典型使用场景]

## 注意事项 / 常见问题
```

### 工作汇报 / 周报

```markdown
## 本周完成
- [具体成果，可量化]

## 进行中
- [任务名]：[当前进度 / 阻塞原因]

## 下周计划
- [优先级排序]

## 风险 / 需支持
- [如有]
```

### 商务邮件结构

1. **主题行**：简明扼要，包含关键词和行动项
   - ❌ 「关于项目的一些问题」
   - ✅ 「[待确认] XX 项目上线时间表 — 需您在 3/15 前回复」

2. **正文**：
   - 开门见山：第一段说明邮件目的
   - 分点列清：每个问题/请求单独一条
   - 明确截止：「请于 XX 前回复/确认」

3. **结尾**：感谢 + 联系方式

### 分析报告结构

```
执行摘要（2-3 段）
  ↓
背景与问题定义
  ↓
方法论 / 数据来源
  ↓
分析发现（用图表 / 数据支撑）
  ↓
结论与建议
  ↓
附录（原始数据、参考文献）
```

## 中文写作常见问题

| 问题 | 错误示例 | 正确写法 |
|------|---------|---------|
| 主语缺失 | 「经过研究，结果表明…」 | 「研究人员发现…」 |
| 被动滥用 | 「该问题被解决了」 | 「团队解决了该问题」 |
| 冗余表达 | 「进行了详细深入的研究」 | 「深入研究了」 |
| 中英混杂 | 「做了一个update」 | 「更新了版本」 |

## 润色 Checklist

完成初稿后逐项检查：
- [ ] 标题能否独立概括内容？
- [ ] 每段是否只有一个核心观点？
- [ ] 数字、日期、名称拼写是否一致？
- [ ] 删去所有"总的来说"、"综上所述"等无意义过渡词
- [ ] 大声读出来是否流畅自然？
